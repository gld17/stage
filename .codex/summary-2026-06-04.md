## IMPL Reviewer Fixes Applied (Round-0 Fix Loop)
1. **grad_updater.py — ValueError on `others.remove(grad)`**
   - Root cause: In VLM, some PlaceHolder weights (e.g., `vision_projection.w`, `patch_embedding.w_patch`) are also gradients, so `grad` is not in `others`.
   - Fix: `if grad in others: others.remove(grad)` (defensive guard).

2. **grad_updater.py — `AttributeError: 'NoneType' object has no attribute 'name'`**
   - Root cause: `get_weights_grads_others()` appends `tensor._grad` to `grads` even when `_grad` is `None`.
   - Fix: Only append if `tensor._grad is not None`.

3. **reshape.py — `TypeError: cannot determine truth value of Relational`**
   - Root cause: ViT Patch Embedding Reshape uses symbolic shapes (`NumPatches * VisionPatchDim` vs `VisionImageSize² * VisionInChannels`) that cannot be numerically compared before symbol values are bound.
   - Fix: Wrap `assert` in `try/except TypeError` to skip symbolic-only sanity checks.

## Round-1: Current Blocker — `KV(VisionHead)` in Chakra Conversion

### Root Cause Analysis
- `vision_encoder.py::_vit_block()` calls `ReplicateGraph.apply()` with:
  ```python
  old_symbol_map_new_symbol={
      "Seq": "NumPatches",
      "Dmodel": "VisionHidden",
      "Dff": "VisionIntermediate",
      "Head": "VisionHead",
      "KVHead": "VisionHead",
  }
  ```
- `ReplicateGraph._update_symbols()` iterates the dict and performs **substring replacement** on `op_attr`:
  ```python
  tensor.op_attr = tensor.op_attr.replace(f"{str(from_)}", f"({str(to_)})")
  ```
- **Bug**: Dictionary order causes `"Head"` to be processed **before** `"KVHead"`.
  - Original `op_attr`: `KVHead/tp`
  - After `"Head" → "VisionHead"` replacement: `KV(VisionHead)/tp`
  - `"KVHead"` is no longer found for its own replacement.
- Result: `op_attr` contains `KV(VisionHead)/tp`, which sympy parses as `KV(VisionHead)` — treating `KV` as a function call on `VisionHead`.
- This is a **framework-level bug** in `_update_symbols` that is triggered by VLM's symbol mapping but could affect any model with overlapping symbol names.

### Fix Direction
Modify `ReplicateGraph._update_symbols()` to process replacements in **descending order of string length** (longest key first), preventing short keys from corrupting longer keys that contain them as substrings.
